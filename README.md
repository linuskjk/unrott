# Unrott (SwiftUI iOS MVP)

Unrott enforces one shared daily screen-time budget across selected apps and unlocks additional time via camera-detected push-ups.

## Target Membership

Add files to targets exactly like this:

- App target (`Unrott`): all files in `UnrottApp` + all files in `UnrottShared`
- Extension target (`ScreenTimeMonitorExtension`): `ScreenTimeMonitorExtension.swift` + all files in `UnrottShared`
- Extension target (`ScreenTimeReportExtension`): `ScreenTimeReportExtension.swift`, `SharedUsageReportScene.swift` + all files in `UnrottShared`

## Xcode Setup (Step-by-Step)

1. Create app target:
- Xcode -> New Project -> iOS App
- Product Name: `Unrott`
- Interface: `SwiftUI`
- Language: `Swift`
- iOS Deployment Target: iOS 16.0+

2. Add extension targets:
- File -> New -> Target -> `Device Activity Monitor Extension`
- Name it `ScreenTimeMonitorExtension`
- File -> New -> Target -> `Device Activity Report Extension`
- Name it `ScreenTimeReportExtension`

3. Add source files:
- Copy all folders/files from this workspace into your Xcode project.
- Ensure target membership matches the mapping above.

4. Configure app group:
- Open `UnrottShared/SharedConstants.swift`
- Replace `group.com.example.unrott` with your real App Group ID.
- In Signing & Capabilities, add `App Groups` capability to all 3 targets and enable the same ID.

5. Configure Family Controls / Screen Time capabilities:
- Add `Family Controls` capability to app target.
- Add `Family Controls` capability to `ScreenTimeMonitorExtension` target.
- Keep Screen Time related entitlements enabled as generated by Xcode templates.

6. Configure camera permission:
- In app target Info, add `Privacy - Camera Usage Description` (`NSCameraUsageDescription`).
- Suggested value: `Camera is used to detect push-ups and unlock extra screen time.`

7. Add frameworks if Xcode did not auto-link:
- App target: `FamilyControls`, `DeviceActivity`, `ManagedSettings`, `Vision`, `AVFoundation`
- Monitor extension: `DeviceActivity`, `ManagedSettings`, `FamilyControls`
- Report extension: `DeviceActivity`, `SwiftUI`

8. Bundle identifiers and signing:
- Set unique Bundle IDs for all targets.
- Use same Team for all targets.
- Build for a real device (not simulator).

## How Blocking Works Internally

1. User picks apps via `FamilyActivityPicker`.
2. `ScreenTimeManager` starts a repeating daily `DeviceActivitySchedule` with one event threshold:
- threshold minutes = `dailyLimitMinutes + earnedMinutesToday`
- event includes all selected applications/categories/domains
3. When threshold is reached, `ScreenTimeMonitorExtension.eventDidReachThreshold` fires in background and:
- marks state as blocked in shared app-group storage
- applies `ManagedSettingsStore.shield` to all selected apps/categories/domains
4. Apps are blocked system-wide.
5. User opens Unrott, performs push-ups, and claims reward.
6. Claimed reward increases `earnedMinutesToday` globally.
7. App clears shield and restarts monitoring with updated threshold.
8. When extra minutes are consumed, threshold is reached again and extension re-applies blocking.

## State Model

Persisted in app-group `UserDefaults` through `SharedStore`:

- `dailyLimitMinutes`
- `earnedMinutesToday`
- `totalUsedMinutesToday`
- `remainingMinutes` (derived)
- `isBlocked`
- selected app tokens (`FamilyActivitySelection`)
- daily reset marker (`lastResetDayIdentifier`)

Daily reset occurs automatically when date changes.

## Push-Up Detection Approach

`PushUpDetector` uses camera frames + Vision body pose (`VNDetectHumanBodyPoseRequest`):

- Computes left/right elbow angles (shoulder-elbow-wrist)
- Uses smoothing window to reduce jitter
- State machine with hysteresis:
  - `up` -> `down` below angle threshold
  - `down` -> `up` above angle threshold
- Requires minimum down-hold and min interval between reps
- Converts reps to reward at `5 reps = 1 minute`

## Test on Real Device

1. Install and launch app on iPhone.
2. On Setup tab:
- Grant Screen Time authorization
- Set limit to a low value (for test, 1-2 min)
- Select social apps (do not select Unrott itself)
3. Use selected apps until limit is reached.
4. Verify selected apps are shielded/blocked.
5. Open Push-Up tab:
- Grant camera permission
- Start camera and perform push-ups
- Confirm live rep count increments
- Claim earned minutes
6. Re-open blocked apps and verify temporary unlock.
7. Continue usage and verify apps block again when earned minutes run out.

## Edge Cases Covered

- Camera permission denied UI path
- No apps selected -> no monitor activation
- Negative values prevented with clamping
- Duplicate push-up counting reduced by smoothing + rep interval + state transitions
- Background enforcement handled by monitor extension

## Notes

- FamilyControls / DeviceActivity workflows require a real iPhone and proper entitlements.
- Simulator does not support full Screen Time enforcement behavior.
- This MVP is intentionally lean and production-structured (MVVM + shared state + extensions), ready for incremental hardening.
